# Dissertation Plot Strategy

## Purpose

This document defines a conservative plot strategy for the MSc dissertation.
It reviews what the current repository can support, which figures are
notebook-aligned recoveries, which are helper-derived analysis figures, and
which figures should wait for calibration or further modelling.

No new plots are generated by this document. It is a planning layer only.

## Current Plotting Boundary

The repository currently supports four different kinds of plot sources:

1. Notebook-aligned recovery outputs under
   `results/notebook_aligned_recovery/`.
2. Deterministic helper outputs from the migrated simulator core.
3. Deterministic Faraday optimisation outputs under
   `results/faraday_optimisation_v1/`.
4. Calibration-readiness observables from absorption / RAI preprocessing
   helpers.

These sources should not be mixed casually. The failed broad figure gallery
showed that valid helper calls can still make visually or scientifically wrong
figures if they bypass notebook defaults, omit display transformations, or
plot a different intermediate quantity.

The dissertation plot layer should therefore use explicit configs, record
metadata, and state whether each figure is:

- a notebook-aligned recovery;
- a Version 1 representative / uncalibrated analysis plot;
- a calibration-readiness demonstration;
- a future calibrated result.

## Plot Families

### 1. Condensate and Column-Density Figures

Status: **READY NOW**

Source:

- `scripts/recover_notebook_condensate_stage.py`
- `scripts/generate_condensate_three_view.py`
- `results/notebook_aligned_recovery/condensate_stage/`
- `results/notebook_aligned_recovery/condensate_three_view/`

Scientific question:

What physical object is being imaged, and what column-density distribution does
the optical model see?

Recommended dissertation use:

- main text: one compact Thomas-Fermi condensate / column-density figure;
- appendix: three-view condensate projection if useful for axis conventions.

Best design:

- image-like maps for column density;
- central lineouts for 3D density or column-density cuts;
- axes in micrometres;
- density in `cm^-3` only for notebook-style recovery figures, or `m^-3` for
  thesis parameter tables;
- column-density maps labelled as `n_col(y,z)`, `n_col(x,z)`, or
  `n_col(x,y)`;
- reserve `n_tilde_x`, `n_tilde_y`, and `n_tilde_z` for peak scalar column
  densities.

Risk:

The main risk is notation, not numerical validity. The subscript on
`n_tilde_i` means integration along axis `i`, not a lineout direction.

Implementation need:

No new numerical support is needed. Future plotting scripts should reuse the
existing recovery outputs rather than recomputing values ad hoc.

### 2. Scalar Phase and Faraday Rotation Maps

Status: **READY NOW**

Source:

- `scripts/recover_notebook_phase_stage.py`
- `scripts/recover_notebook_faraday_stage.py`
- `scalar_phase_shift(...)`
- `faraday_rotation_angle(...)`

Scientific question:

How does the condensate density become the phase or rotation signal that
non-destructive imaging measures?

Recommended dissertation use:

- main text: one scalar phase map or lineout after the condensate figure;
- main text or Faraday chapter: Faraday rotation map only if clearly labelled
  as `theta_F = kappa_F * phi` with `kappa_F = 1.0`;
- appendix: additional lineouts or comparison statistics.

Best design:

- image map plus central lineout;
- phase in radians;
- fixed notebook-aligned extent and zoom;
- avoid normalising phase maps to their maximum unless the caption explicitly
  says the plot is shape-only.

Risk:

The Faraday rotation is phenomenological in Version 1. Captions must not imply
that `kappa_F` is experimentally fitted or microscopically derived.

Implementation need:

No new numerical support is needed for current notebook-aligned maps. A future
analysis plot of phase versus detuning needs only a small plotting script
calling existing light-atom helpers.

### 3. PCI, DGI, and Faraday Ideal Image Comparison

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- `simulate_pci_image(...)`
- `simulate_dgi_image(...)`
- `simulate_faraday_image(...)`
- notebook-aligned recovery outputs in `pci_stage/`, `dgi_stage/`, and
  `faraday_stage/`

Scientific question:

How do PCI, DGI, and Faraday convert the same condensate signal into different
ideal image observables?

Recommended dissertation use:

- main text: one carefully controlled comparison figure;
- appendix: separate ideal image panels and lineouts for each mode.

Best design:

- use one fixed condensate, imaging axis, grid, detuning, and display window;
- use separate panels, not a single overlaid image;
- preserve physical units:
  - PCI and DGI: `I/I0`;
  - Faraday dark-field: dark-field intensity;
  - Faraday dual-port: normalised signal `S`;
- do not rescale all images to `0..1` for visual convenience;
- include lineouts if the colour scales make cross-mode comparison hard.

Risk:

Cross-mode images have different physical meanings and dynamic ranges. A
single shared colour scale is usually inappropriate. Normalising every panel
can make the comparison visually attractive but scientifically misleading.

Implementation need:

The individual notebook-aligned stages are already recovered. A small
orchestration script could assemble a dissertation comparison figure from the
existing stage outputs and metadata without changing simulator code.

### 4. Camera and Noise Demonstration

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- `simulate_camera_image(...)`
- `simulate_noisy_camera_image(...)`
- recovered deterministic and stochastic camera stages

Scientific question:

How does a continuous image become a finite-pixel, finite-photon measurement?

Recommended dissertation use:

- methods chapter: deterministic camera binning and one noisy frame;
- appendix: detailed noise statistics and central lineout comparison.

Best design:

- two panels: noise-free camera image and one explicit-seed noisy frame;
- optional third panel: central lineout or residual/noise distribution;
- axes in camera-plane object-space micrometres if using the notebook
  recovery convention;
- image units `I/I0`;
- state the photon budget and read noise in the caption.

Risk:

Exact notebook-global RNG replay is not guaranteed after arbitrary interactive
notebook execution. Dissertation figures should state that stochastic examples
use an explicit seed and represent the notebook noise model, not a unique
historical notebook pixel pattern.

Implementation need:

Existing outputs are sufficient for a basic demonstration. A cleaner
presentation figure would need a small plotting script that reads the recovered
camera JSON/CSV outputs and applies the standard figure-language conventions.

### 5. Deterministic Multishot Sequence

Status: **READY NOW**

Source:

- `simulate_multishot_sequence(...)`
- `accumulate_snr(...)`
- `scripts/recover_notebook_multishot_stage.py`
- `results/notebook_aligned_recovery/multishot_stage/`

Scientific question:

How does repeated imaging trade accumulated information against condensate
loss or heating?

Recommended dissertation use:

- main text: one multishot sequence plot;
- appendix: full sequence table or clean-loss versus heating comparison.

Best design:

- four compact panels:
  - condensate depletion versus frame index;
  - peak phase versus frame index;
  - per-frame SNR versus frame index;
  - accumulated SNR versus frame index;
- frame index on the x axis, not time unless actual elapsed time is plotted;
- include both clean-loss and heating models only if the caption explains the
  distinction;
- use linear axes for the current recovered sequence.

Risk:

The deterministic sequence is not the same as a noisy filmstrip. It describes
state evolution and expected signal/SNR, not the exact observed frame content.

Implementation need:

Already supported. Future improvements are presentation-level only.

### 6. Noisy PCI Multishot Filmstrip

Status: **WAIT FOR VISUAL REVIEW OF CURRENT FIGURES**

Source:

- `scripts/recover_notebook_noisy_multishot_filmstrip.py`
- `results/notebook_aligned_recovery/noisy_multishot_filmstrip/`

Scientific question:

What does the condensate look like across a noisy repeated PCI measurement
sequence?

Recommended dissertation use:

- main text only if visually clear at print size;
- otherwise appendix or repository results.

Best design:

- selected frame indices only, not every frame;
- consistent colour scale across frames;
- frame labels should include frame index, atom number or loss fraction, and
  possibly accumulated SNR if readable;
- keep stochastic caveat in caption.

Risk:

Filmstrips can be visually persuasive but can overstate stochastic
reproducibility. They should not be used as quantitative evidence without
paired sequence plots or statistics.

Implementation need:

The current recovered figure exists. Before using it in the dissertation, it
should be visually reviewed at final page size and compared against the
deterministic multishot sequence.

### 7. Detuning Physics and Destructiveness Trade-Offs

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- `dimensionless_detuning(...)`
- `scalar_phase_shift(...)`
- `residual_optical_depth(...)`
- `scattered_photons_per_atom(...)`
- `reabsorption_fraction(...)`
- `faraday_rotation_angle(...)`
- `sweep_faraday_detuning(...)`

Scientific question:

Why does operating farther from resonance reduce destructiveness faster than it
reduces dispersive signal, and where does useful information per destructive
event become favourable?

Recommended dissertation use:

- main text: one detuning trade-off figure;
- methods or appendix: underlying phase/scattering scaling curves.

Best design:

- x axis: absolute detuning, preferably `Delta / 2pi` or detuning in GHz as
  used by the notebook; use a logarithmic x axis if spanning more than roughly
  one decade;
- panel A: phase-like signal or Faraday rotation magnitude versus detuning;
- panel B: scattered photons per atom and/or residual OD versus detuning;
- panel C, optional: `signal_per_scattered_photon` or
  `information_per_scattered_photon`;
- use log y axis for scattering/destructiveness if values span orders of
  magnitude;
- avoid plotting positive and negative detuning together unless the sign
  convention is explicitly discussed.

Risk:

The current optimisation helpers are deterministic and uncalibrated. The best
detuning from `signal_per_scattered_photon` is not a final experimental
operating point because camera noise, finite signal floor, `kappa_F`
calibration, and experimental constraints are not yet fitted.

Implementation need:

A small script should be enough. It should use `configs/notebook_v1_defaults`
or a dedicated plot config, not hard-coded parameters.

### 8. Real and Imaginary Susceptibility-Style Curves

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- existing scalar light-atom formulas:
  - dispersive part proportional to `delta / (1 + delta^2)`;
  - absorptive part proportional to `1 / (1 + delta^2)`.

Scientific question:

Why do phase imaging and destructiveness scale differently with detuning?

Recommended dissertation use:

- physics background or methods chapter;
- not a headline result.

Best design:

- x axis: dimensionless detuning `delta` or detuning in linewidth units;
- y axis: normalised response, not calibrated image signal;
- two curves:
  - dispersive response;
  - absorptive response;
- use a symmetric x range only if both red and blue detuning signs are being
  explained;
- otherwise use positive detuning and focus on magnitude.

Risk:

This is an explanatory response plot, not a full vector polarizability model
for erbium. Captions should say it illustrates the scalar Version 1 line-shape
convention.

Implementation need:

Only plotting/config support is needed. Do not introduce new physics helpers
unless repeated use justifies a small analysis utility.

### 9. Faraday Single-Variable Optimisation Sweeps

Status: **READY NOW, BUT PRESENT AS REPRESENTATIVE V1**

Source:

- `evaluate_faraday_operating_point(...)`
- `sweep_faraday_detuning(...)`
- `sweep_faraday_intensity(...)`
- `sweep_faraday_exposure_time(...)`
- `summarise_faraday_sweep(...)`
- `scripts/generate_dissertation_results.py`
- `results/faraday_optimisation_v1/`

Scientific question:

How do detuning, probe power, and exposure time change the deterministic
information-versus-destruction metrics?

Recommended dissertation use:

- main text: one summary figure or table after the physical trade-off figure;
- appendix: individual sweep tables.

Best design:

- keep the current generated tables as the quantitative source;
- revise figures only after visual review if needed;
- for detuning:
  - x axis in GHz, likely log or semilog if the sampled range grows;
  - y axis should show the selected metric and destructiveness separately;
- for probe power and exposure time:
  - linear x axes are acceptable for the current small ranges;
  - avoid implying a global optimum if the metric is flat or monotonic;
- include the selected metric name explicitly.

Risk:

The current representative results use placeholder values and `kappa_F = 1.0`.
They are useful for demonstrating workflow, not for claiming a final
experimental optimum.

Implementation need:

Already generated for V1 representative results. Any dissertation-ready update
should improve figure presentation and metadata but not change helper
behaviour.

### 10. Signal Per Scattered Photon and Information Efficiency

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- Faraday optimisation helpers;
- light-atom scattering helper;
- possibly multishot sequence outputs.

Scientific question:

Which operating choices give the most useful signal per destructive event?

Recommended dissertation use:

- central result concept, but with careful caveats;
- should accompany rather than replace raw signal and scattering curves.

Best design:

- plot `signal_per_scattered_photon` or
  `information_per_scattered_photon` against detuning;
- optionally show raw signal and scattering in adjacent panels;
- do not use this ratio alone if scattered photons approach zero and the ratio
  becomes dominated by asymptotic scaling;
- mark the sampled operating range and any imposed minimum signal or camera
  SNR threshold.

Risk:

Ratios can over-favour very weak signals at very large detuning. The plot is
scientifically useful only when paired with a minimum detectability condition
or a raw-signal panel.

Implementation need:

Small plotting/orchestration support only. A future objective function may need
a minimum-SNR constraint, but that should be a separate analysis milestone.

### 11. Camera Photon Budget and Noise Scaling

Status: **READY AFTER SMALL ORCHESTRATION SCRIPT**

Source:

- `intensity_at_atoms(...)`
- camera recovery summaries;
- `simulate_camera_image(...)`
- `simulate_noisy_camera_image(...)`

Scientific question:

When is the measurement photon-shot-noise limited, and when does read noise
matter?

Recommended dissertation use:

- methods or appendix;
- main text only if camera noise is central to the optimisation claim.

Best design:

- x axis: photons per camera pixel or exposure time;
- y axis: expected fractional noise, SNR proxy, or residual standard
  deviation;
- use log-log axes if spanning decades;
- overlay photon shot-noise scaling and read-noise floor;
- optionally add one explicit-seed simulation point with error band from
  repeated seeds in future work.

Risk:

Current stochastic support is explicit-seed single-frame simulation, not
noise-averaged optimisation. Avoid claiming stochastic robustness unless an
averaging study is added.

Implementation need:

Small plotting support is enough for deterministic noise-budget curves.
Stochastic bands require an explicit repeated-seed workflow and should be a
separate milestone.

### 12. Calibration-Readiness Plots

Status: **WAIT FOR EXPERIMENTAL CALIBRATION**

Source:

- `compute_optical_density(...)`
- `integrate_optical_density(...)`
- `estimate_cloud_moments(...)`
- `extract_absorption_observables(...)`

Scientific question:

How will destructive absorption / RAI data calibrate density, optical depth,
camera, and eventually Faraday parameters?

Recommended dissertation use:

- methods/future-work chapter now;
- results chapter only after real or approved synthetic calibration data is
  introduced.

Best design after data exists:

- OD image with ROI;
- horizontal and vertical OD lineouts;
- fitted centre and widths;
- table of extracted observables;
- residual plot if fitting is implemented;
- calibration before/after comparison for simulator parameters.

Risk:

Synthetic calibration plots can be mistaken for experimental calibration.
Until lab data is available, keep these as workflow schematics or clearly
labelled synthetic examples.

Implementation need:

Real-data loaders, calibration config schema, and fitting are still missing.
Do not generate experimental-looking calibration figures from arbitrary
synthetic arrays without strong labelling.

### 13. Two-Dimensional Operating Maps

Status: **WAIT FOR FUTURE MODEL EXTENSION**

Source:

- not fully implemented in the migrated helper layer;
- original notebook contains operating-map logic, but it is not yet recovered
  as a canonical plot recipe.

Scientific question:

Where are viable operating regions in two-parameter space, for example
detuning versus exposure time or probe power versus exposure time?

Recommended dissertation use:

- future work, unless a dedicated recovery/validation milestone is completed;
- appendix only if explicitly labelled as exploratory.

Best design:

- heatmap with one scalar objective;
- contour overlays for constraints:
  - maximum scattered photons;
  - minimum per-frame SNR;
  - maximum condensate loss;
- separate panels for raw signal, destructiveness, and objective.

Risk:

A heatmap looks like a final recommendation even when the model is
uncalibrated. Without fitted camera and `kappa_F` parameters, it can
overclaim.

Implementation need:

Requires a new deterministic two-dimensional sweep helper or script, explicit
constraints, and careful visual review. It should not be generated as the next
step.

### 14. Beyond-Thomas-Fermi and Mixture Plots

Status: **WAIT FOR FUTURE MODEL EXTENSION**

Source:

- documented in `docs/extension_roadmap.md`;
- no current state generators for droplets, supersolids, or mixtures are part
  of the Version 1 migrated core.

Scientific question:

How would non-Thomas-Fermi density distributions or multiple components change
the imaging problem?

Recommended dissertation use:

- future-work schematic only;
- no quantitative result figure in the current Version 1 dissertation unless a
  future model-extension milestone is completed.

Best design after implementation:

- compare state density maps;
- compare corresponding phase or Faraday signal maps;
- include clear labels that the state generator is phenomenological or
  externally loaded.

Risk:

Inventing unsupported density shapes for the dissertation would undermine the
notebook-aligned validation story.

Implementation need:

New additive state/profile generators or external density-map input support.

## Plotting Roadmap

### Stage 1: Stabilise Existing Notebook-Aligned Figures

Use existing recovery outputs and standardised figure language. Do not broaden
the gallery. Select only the figures that support the dissertation narrative:

- condensate / column density;
- scalar phase;
- ideal PCI/DGI/Faraday examples;
- deterministic multishot sequence.

### Stage 2: Add One Controlled Imaging Comparison Figure

Create a small script that assembles PCI, DGI, and Faraday ideal outputs from
existing recovered arrays and metadata. The script should not recompute new
physics unless it calls the existing recovery stage builders with the canonical
config.

Required metadata:

- config path and commit hash;
- stage sources used;
- units and normalisation for each panel;
- statement that outputs are notebook-aligned Version 1 and uncalibrated.

### Stage 3: Add Physics Trade-Off Curves

Create one deterministic plotting script for:

- phase or Faraday signal versus detuning;
- scattered photons / residual OD versus detuning;
- signal per scattered photon versus detuning.

This is the strongest next analytical plot because it directly supports the
core research question.

### Stage 4: Refine Faraday Optimisation Result Plots

Keep the current CSV/JSON tables as the quantitative source. Improve only the
figure presentation if needed:

- clearer axis labels;
- explicit representative V1 / uncalibrated status;
- separate metric and destructiveness axes or panels;
- no claim of final optimum.

### Stage 5: Calibration-Facing Plots After Data Arrives

After absorption / RAI data exists:

- OD image and lineouts;
- extracted cloud moments;
- fitted parameter table;
- before/after calibrated simulation comparison;
- held-out validation residuals.

### Stage 6: Future Operating Maps

Only after deterministic single-variable plots are accepted:

- add 2D sweep tables;
- define constraints first;
- then plot heatmaps with contour overlays;
- keep calibrated and uncalibrated maps separate.

## Metadata Policy

Every future dissertation plot script should write metadata containing:

- source script;
- config file;
- git commit hash;
- helper functions or recovery stages used;
- whether the plot is notebook-aligned, representative V1, calibrated, or
  future-model;
- physical units and normalisation;
- `kappa_F` value if Faraday appears;
- calibration status;
- stochastic seed policy if noise appears.

## Final Recommendation

The next implementation should not be another broad figure gallery.

The best next step is a narrow dissertation plot layer that does one of two
things:

1. assembles a controlled PCI/DGI/Faraday comparison from existing
   notebook-aligned recovery outputs; or
2. generates a dedicated detuning trade-off figure showing signal,
   scattering/destructiveness, and signal per scattered photon.

Of these, the detuning trade-off figure is the highest-value new analysis plot
because it directly supports the dissertation's information-versus-destruction
argument while using helpers that are already tested.
